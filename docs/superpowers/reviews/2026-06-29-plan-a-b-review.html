3. Auth System — thiếu edge cases

Missing Impact
Refresh token flow User phải login lại khi token hết hạn
Force logout (admin revoke) Admin cần revoke token của user bị khoá
Session cleanup cron Session hết hạn tích tụ → performance issue
Password change flow Owner đổi password của staff

4. ⚠️ Task 3.1 — Tenant creation race condition

Tạo tenant + owner + branch + botConfig trong 1 API call nhưng không có transaction. Nếu step 4 fail → inconsistent state.

1. Insert Tenant
2. Insert User (role=owner)
3. Insert Branch
4. Insert BotConfig ← FAIL HERE
5. Insert default Roles
6. AuditLog
❌ Inconsistent: có Tenant, Owner nhưng không có BotConfig

Fix: Wrap trong Turso transaction

5. Order — thiếu inventory check

Task 3.7 tạo Order nhưng không check inventory cho type='product'. Khi bán sản phẩm (mỹ phẩm, kem dưỡng) → phải trừ tồn kho.

6. Booking — spec vs plan không align

Spec Section 3.6.1 nói: "Khi completed → tự tạo Order tương ứng". Nhưng Plan A Task 3.8 KHÔNG đề cập logic này.

7. Missing: Customer lookup by phone

Nhiều flow cần lookup customer bằng phone (Zalo message, Web widget). Plan A có index nhưng không có API/service để lookup.

2. ⚠️ Task 1.2 — Context builder thiếu PII Masking

🔴

Vi phạm data-privacy.mdc

Plan B build context nhưng không mask PII. Mọi PII phải được mask trước khi gửi lên LLM.

import { maskPII } from '@/lib/chat/masking'

// Trong buildAIContext:
const maskedProducts = productList.map(p => ({
  ...p,
  name: maskPII(p.name)
}))

3. Task 1.3 — Intent detection quá đơn giản

Plan B dùng regex đơn giản. Vietnamese có nhiều cách diễn đạt hơn:

// Phải cover:
"cho tôi nói chuyện với người thật"
"bot ngu quá"
"human đâu"
"tôi cần tư vấn thật"

4. Task 2.3 — Widget script có vấn đề

  • Memory leak: cần cleanup khi unload
  • Multiple widgets: cần check và reject
  • XSS vulnerability: cần verify origin

5. Task 3.1 — Zalo reconnect logic có bug

Khi Next.js hot-reload hoặc connection die → cần periodic health check:

setInterval(async () => {
  for (const [tenantId, instance] of zaloInstances) {
    const isAlive = await checkZaloConnection(instance.api)
    if (!isAlive) {
      await reconnectZalo(tenantId)
    }
  }
}, 5 * 60 * 1000)

6. Task 3.4 — Missing: Zalo webhook verification

zca-js có webhook feature nhưng Plan B không đề cập: message read, user typing, etc.

⚠️ Cảnh Báo Pháp Lý: Zalo Cookie

zca-js là unofficial/non-public API. Zalo có Terms of Service cấm scraping/automation. Dùng cookies để login có thể vi phạm.

Rủi ro: Account bị ban, legal action từ VNG

Đề xuất:

  • Tham khảo legal counsel trước khi deploy
  • Có fallback: dùng Zalo Official Account (ZNS) thay vì zca-js
  • Document disclaimer: "Sử dụng unofficial API, tự chịu rủi ro"

📊 Performance — Chat Message Volume

Spec giả định: Chat volume < 1,000 messages/ngày per tenant

Thực tế: 1,000 messages/ngày = ~0.7 messages/phút. Nếu peak (sale campaign): có thể 10,000+ messages/ngày.

Caching strategy cần có:

// Cache AI response cho cùng query trong 5 phút
const cache = new LRUCache<string, string>({ max: 1000 })
