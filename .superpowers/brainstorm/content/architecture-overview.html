Current Architecture vs Proposed Architecture

Chuyển từ SQLite local + Google Sheets sang Turso (SQLite at the edge)

Current Architecture
Frontend (Next.js)
SQLite local (dev only)
↓ HTTP
n8n (VPS)
Google Sheets
↓ Read/Write
Google Sheets
DatCho, LogChat, etc.
❌ Vercel không kết nối được SQLite file
❌ Google Sheets API chậm, rate limit
Proposed: Turso Architecture
Vercel
Next.js (Frontend)
↓ LibSQL / Prisma
Turso
SQLite at the Edge
↓ LibSQL
n8n (VPS)
Kết nối Turso qua HTTP
✅ Turso dùng SQLite → chuyển dễ dàng
✅ Vercel kết nối trực tiếp
✅ n8n dùng HTTP Request

2 Options để n8n kết nối Turso

A

Option A: n8n gọi API qua HTTP

Frontend tạo API endpoints (Next.js API routes), n8n gọi qua HTTP Request node. Đơn giản, không cần Turso client trong n8n.

B

Option B: n8n cài Turso CLI

n8n chạy command `turso db exec` hoặc dùng HTTP client để query trực tiếp. Nhanh hơn nhưng cần config hơi phức tạp.
